Request-flyt

Offentlig innsjekk (mobil)

Mobil scanner QR
   → GET /checkin/[qrToken]    (Server Component — henter dør + ansatte)
   → POST /api/visits          (Oppretter Visit, sender e-post)
   → Redirect til /checkin/confirmation

Ansatt logger inn

POST /api/auth/signin          (NextAuth Credentials)
   → JWT-token med id + rolle
   → proxy.ts tillater tilgang til beskyttede ruter

Besøkslogg med redigering

GET /visits                    (Server Component — Prisma-spørring med filter)
PATCH /api/visits/[id]         (Auth-sjekk → oppdater Visit + skriv VisitAuditLog)

Autentisering og tilgangskontroll

Alle forespørsler
   ↓
proxy.ts (Edge Runtime)
   ├── /checkin/*, /checkout/*, /login, /terms  → Tillatt uten innlogging
   └── /dashboard/*, /visits/*, /admin/*        → Krev JWT-sesjon
                                                    → Ingen sesjon → redirect /login

API-ruter
   └── auth() sjekkes per handler
       ├── 401 hvis ikke innlogget
       └── 403 hvis feil rolle (f.eks. GDPR krever ADMIN)
